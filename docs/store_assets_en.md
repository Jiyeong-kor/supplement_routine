# Store Assets

## Goal

Supplement Routine store assets should communicate a user-defined routine manager, not a medical product.

## Required Assets

| Asset | Recommended Content |
| --- | --- |
| App icon | Keep the current launcher icon |
| Feature graphic | App name, today's progress, record-focused message |
| Phone screenshots | Today, add supplement, history, settings |
| Tablet screenshots | Wide layout with NavigationRail |
| README media | 2-4 screenshots and one short usage-flow GIF |

## Copy Direction

- Prefer: `today routine`, `intake records`, `user-entered schedule`
- Avoid: `treatment`, `prescription`, `efficacy`, `recommendation`

## Output Paths

- Store graphics: `docs/assets/store/`
- README screenshots: `docs/assets/screenshots/`
- README GIFs: `docs/assets/demos/`

## Capture Checklist

1. Verify both light and dark modes.
2. Capture with representative mock data.
3. Avoid text that looks like personal information.
4. Capture notifications and the home widget separately from the main app screens.
