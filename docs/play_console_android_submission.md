# Android Play Console 제출 준비

이 문서는 Supplement Routine Android 우선 출시를 위해 Play Console에 입력할 값을 정리한다. 실제 계정 인증, 정책 동의, 결제, 최종 제출은 계정 소유자가 직접 진행한다.

## 제출 후보

최신 Android 제출 후보:

```text
GitHub Actions `KMP Release` run `27008729353`의 `kmp-android-release` artifact 안에 있는 `bundle/release/androidApp-release.aab`
```

검증:

- `KMP Release` run `27008729353`, `platform=android` 성공
- `androidApp-release.aab` 13,252,323 bytes
- `jarsigner -verify` 결과 `jar verified`
- `androidApp-release.apk` 13,686,333 bytes
- `apksigner verify --verbose --print-certs` 결과 `Verifies`, v1/v2 signing true
- APK 인증서 SHA-256 digest: `f90457e364fc0c14d30abdd841efadc8d7ba69b55cff9455cfcf63cb6afe83c2`

## 스토어 등록정보 초안

### 짧은 설명

직접 정한 영양제 복용 루틴을 확인하고 기록하는 local-first 루틴 앱

### 긴 설명

Supplement Routine은 사용자가 직접 정한 영양제 복용 루틴을 관리하는 앱입니다.

오늘 어떤 영양제를 복용해야 하는지 확인하고, 복용 후 바로 체크해 기록을 남길 수 있습니다. 식사 기준, 특정 시각, 일정 간격 같은 복용 규칙을 입력하면 오늘 일정과 최근 기록을 한곳에서 볼 수 있습니다.

주요 기능:

- 오늘 복용 일정 확인
- 복용 완료 체크와 체크 해제
- 영양제 이름, 복용량, 복용 방식, 메모 등록
- 오늘 완료율과 최근 기록 확인
- 식사 시간과 알림 기본값 설정
- 로컬 알림과 정확한 알림 권한 안내
- 기기 안에 저장되는 local-first 데이터 관리

Supplement Routine은 영양제 추천, 효능 설명, 의료 조언, 진단, 처방 기능을 제공하지 않습니다. 영양제 복용과 관련된 결정은 전문가와 상담해야 합니다.

## 데이터 보안 답변 기준

현재 Android 앱 기준:

| 질문 범위 | 답변 기준 |
| --- | --- |
| 앱이 사용자 데이터를 수집하나요? | 아니요. 개발자 서버로 전송하거나 제3자 SDK에 제공하지 않습니다. |
| 앱이 사용자 데이터를 공유하나요? | 아니요. |
| 데이터가 기기 밖으로 전송되나요? | 아니요. |
| 계정 생성이 필요한가요? | 아니요. |
| 사용자가 데이터 삭제를 요청할 수 있나요? | 앱 설정의 데이터 초기화 또는 기기 앱 데이터 삭제로 로컬 데이터를 삭제할 수 있습니다. |
| 광고 ID를 사용하나요? | 아니요. |
| 분석 SDK를 사용하나요? | 아니요. |
| 결제/구독을 사용하나요? | 아니요. |

Play Console 문항이 "기기 안에 저장되는 앱 데이터"까지 포함해 묻는 경우에는 영양제 이름, 복용량, 복용 시간, 복용 기록, 메모가 사용자가 직접 입력하는 건강/생활 루틴 정보일 수 있음을 보수적으로 표시한다. 단, 외부 수집/공유는 하지 않는다.

## 콘텐츠 등급 답변 기준

- 폭력, 성적 콘텐츠, 혐오 표현, 도박, 약물 판매, 사용자 생성 콘텐츠: 없음
- 의료 조언, 진단, 치료, 처방: 제공하지 않음
- 건강 관련 입력: 사용자가 직접 정한 루틴 기록 목적
- 위치 정보: 사용하지 않음
- 앱 내 구매: 없음
- 광고: 없음

## 업로드 순서

1. Play Console에서 새 앱을 만든다.
2. 스토어 등록정보에 앱 이름, 짧은 설명, 긴 설명, 스크린샷, 아이콘, feature graphic을 등록한다.
3. 개인정보처리방침 URL을 입력한다.
4. 데이터 보안 설문을 위 기준으로 작성한다.
5. 콘텐츠 등급 설문을 작성한다.
6. 앱 액세스, 타겟층, 뉴스 앱 여부, 광고 여부 등 정책 항목을 작성한다.
7. 내부 테스트 또는 프로덕션 release에 최신 AAB를 업로드한다.
8. release note를 입력한다.
9. 경고 문구를 확인한 뒤 심사 제출한다.
